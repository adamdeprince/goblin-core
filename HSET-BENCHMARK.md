# Goblin Core HSET Speed and Memory Benchmark

Generated by the native C++ harness on a dedicated benchmark host at 2026-07-16T19:07:13Z.

## Summary

At 1,000,000 fields, the leanest Goblin row uses `48.67` RSS bytes/field, `20.1%` less than Redis 8.8 and `45.0%` less than Redis 7.2.4. The Goblin rows beat the fastest incumbent in `7` of `8` existing-field small-hash HSET comparisons. Goblin wins the million-field construction race: `goblin-rt-resp-uds` is `9.7%` ahead of `dragonfly`. RT mixed p99.9 is `16.14 us`, versus `16.71-22.71 us` for the incumbents and `2293.39 us` for efficient Goblin.

## Method

- Every scenario starts a fresh server; engines run one at a time.
- Linux affinity: server core `2`, client/load-generator core `3` (`-1` means unpinned).
- Before the empty-server baseline, every engine constructs and deletes a `2,048`-field fixed-width hash, then settles.
- RT's fixed prefaulted index pool is charged back into RSS and `used_memory` deltas instead of being hidden as empty-server overhead.
- RSS is read from the launched server PID: mini-redis-go uses `ps -o rss=`, while other engines use `/proc/<pid>/status` as `VmRSS + HugetlbPages`; no server-reported RSS field is used.
- Every engine uses RESP2 over a Unix-domain socket. The HSET suite has no TCP, SBE, or shared-memory-ring rows; transport is not a benchmark variable.
- Bulk workloads, response validation, point throughput, mixed latency, process control, aggregation, and report generation run in this native C++ executable.
- The benchmark client's application receive buffer is `65,536` bytes. Goblin's per-client application receive buffer is explicitly `16,384` bytes; incumbent server internals and kernel UDS buffers remain at their engine/OS defaults.
- Bulk loads use multi-field `HSET` batches of `128`. Every point-throughput row uses RESP pipeline depth `256`. Timing includes native client encoding. Point rates use `500,000` requests and the median of `3` probes in each of `3` fresh servers.
- Redis and Valkey use `benchmarks/redis-parity.conf`; Dragonfly uses one proactor thread for single-core parity. mini-redis-go runs as an external binary with `GOMAXPROCS=1`, AOF disabled, and metrics disabled.
- Unsupported point commands are reported as `n/a`; error replies are never timed as successful operations.
- `goblin-efficient-*` uses the default memory-efficient hash representation. `goblin-rt-*` passes `--hash-implementation rt`; the keyspace index remains unchanged so the rows isolate the HSET implementation.
- Goblin uses default string encoding and its configured compact-hash policy. Incumbents are exercised strictly as black-box RESP servers.
