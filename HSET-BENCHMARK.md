# Goblin Core HSET Speed and Memory Benchmark

Generated by the native C++ harness on a dedicated benchmark host at 2026-07-14T04:22:30Z.

## Summary

At one million fields, the leaner Goblin transport uses `48.12` RSS bytes/field, `21.0%` less than Redis 8.8 and `45.7%` less than Redis 7.2.4. The Goblin TCP and ring rows beat the fastest incumbent in `8` of `8` existing-field small-hash HSET comparisons. Goblin loses the million-field construction race: its faster `goblin-sbe-ring` row is `25.1%` behind `dragonfly`. The remaining weakness is insertion-compaction tail latency: Goblin's mixed p99.9 is `2427.87-2457.32 us`, versus `26.28-108.47 us` for the incumbents.

## Method

- Every scenario starts a fresh server; engines run one at a time.
- Linux affinity: server core `2`, client/load-generator core `3` (`-1` means unpinned). Ring client and server must not share a core.
- Before the empty-server baseline, every engine constructs and deletes a `2,048`-field fixed-width hash, then settles.
- RSS is read from the launched server PID: mini-redis-go uses `ps -o rss=`, while other engines use `/proc/<pid>/status` as `VmRSS + HugetlbPages`; no server-reported RSS field is used.
- Transport matrix: Goblin Core is measured over RESP/TCP and typed SBE/shared-memory ring; every incumbent is measured over RESP/TCP. No UDS row is included.
- Bulk workloads, response validation, mixed latency, process control, aggregation, and report generation run in this native C++ executable. TCP point probes use `redis-benchmark`; ring point probes use the typed C++ SBE client.
- Bulk loads use multi-field `HSET` batches of `128`. RESP/TCP uses pipeline depth `256`; SBE/ring currently keeps one request outstanding. Timing includes native client encoding. Point rates use `500,000` requests and the median of `3` probes in each of `3` fresh servers.
- Redis and Valkey use `benchmarks/redis-parity.conf`; Dragonfly uses one proactor thread for single-core parity. mini-redis-go runs as an external binary with `GOMAXPROCS=1`, AOF disabled, and metrics disabled.
- Goblin uses default string encoding and its configured compact-hash policy. Incumbents are exercised strictly as black-box RESP servers.
