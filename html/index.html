The low-latency wire

One cache-line handoff. No syscall. No ASCII.

On Linux, pin client and server to separate cores. The client writes a typed SBE request into shared memory; Goblin Core busy-polls, executes it, and returns the reply through the companion ring.

What the number means

About 220 ns is a full, synchronous, unpipelined request and reply for a one-field HSET or one-member ZADD on a pinned Threadripper PRO 5995WX.

What it costs

The server busy-polls and owns a core. Linux core isolation provides the reliable low-latency deployment; macOS is useful for development but cannot offer the same scheduler control.

Direction, stated plainly

Redis semantics, without the old overhead.

Goblin Core is growing toward the full Redis command surface, implemented with tighter memory layouts and faster execution paths. Today it already covers the core data and scripting workflows, with lists and Pub/Sub arriving July 20, 2026.

The deliberate boundary is infrastructure policy: durable write logs belong in Kafka, where logging is the product. Native Kafka consumption for durable ZADD/ZREM replay arrives July 27, 2026. Use GOBLIN.SAVE/GOBLIN.LOAD for fast point-in-time restarts.

Keys and string values are limited to 64 KiB. Larger values belong in purpose-built blob storage; Goblin Store provides a fast, cost-effective path for those objects.

Available now

  • Sorted sets, hashes, and strings
  • TTL keys and counters
  • Native atomic idioms
  • JavaScript, Python, and more
  • Explicit save/load snapshots

Deliberate boundaries

  • No append-only log
  • No built-in replication
  • No cluster mode
  • Kafka for durable replay
  • Save/load for fast restarts

Source-only release

Build it. Measure it.

Goblin Core is Apache 2.0 licensed C++23. Start with the server over RESP, then opt into shared-memory rings only where owning the core makes sense.

~/goblin-core
$ git clone https://github.com/adamdeprince/goblin-core
$ cd goblin-core
$ cmake -S . -B build-release -DCMAKE_BUILD_TYPE=Release
$ cmake --build build-release
$ ctest --test-dir build-release --output-on-failure
$ ./build-release/goblin-core --port 6379
